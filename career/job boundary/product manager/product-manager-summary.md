# Product Manager (PM)

## 1. Core Responsibility

Product Manager owns:

-   WHAT to build
-   WHY to build it
-   Business value
-   Feature scope
-   Priority

PM is responsible for problem definition and value creation.

------------------------------------------------------------------------

## 2. What Product Manager Must Master

### 2.1 Problem Framing

-   Identify real user pain points
-   Distinguish symptom vs root problem
-   Define measurable goals

### 2.2 Business & Domain Understanding

-   Industry rules
-   Revenue model
-   Risk and compliance awareness
-   Competitive landscape

### 2.3 Requirement Structuring (PRD Writing)

-   Clear functional requirements
-   Edge cases
-   Acceptance criteria
-   Defined scope (in / out)

### 2.4 Prioritization & Tradeoffs

-   Business impact vs engineering cost
-   Risk vs speed
-   Short-term vs long-term value

### 2.5 Cross-Functional Communication

-   Align engineering
-   Align stakeholders
-   Manage expectation

------------------------------------------------------------------------

## 3. What Product Manager Does NOT Own

-   Technical architecture
-   Code implementation
-   Deployment automation
-   Infrastructure management

PM defines direction, not implementation.

------------------------------------------------------------------------

## 4. Boundary Summary

Product Manager owns:

> WHAT & WHY

Engineering owns:

> HOW

Project Manager owns:

> WHEN & WHO
